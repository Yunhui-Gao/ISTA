Preliminaries

Proximal gradient method

Consider the following convex optimization problem:
minx{J(x)≡F(x)+R(x)},(1)\mathop{\mathrm{min}}\limits_{\mathbf{x}} \left\{ J(\mathbf{x}) \equiv F(\mathbf{x}) + R(\mathbf{x}) \right\}, \tag{1}xmin​{J(x)≡F(x)+R(x)},(1)
where F(x)F(\mathbf{x})F(x) and R(x)R(\mathbf{x})R(x) are both convex functions, but different in that F(x)F(\mathbf{x})F(x) is smooth whereas R(x)R(\mathbf{x})R(x) is not. One common approach to problem (1)(1)(1) is the proximal gradient method, which adopts an updating rule given by1
xk+1=proxγkR(xk−γk∇F(xk)),(2)\mathbf{x}_{k+1} = \mathrm{prox}_{\gamma_k R} \left( \mathbf{x}_k - \gamma_k \nabla F(\mathbf{x}_k) \right), \tag{2}xk+1​=proxγk​R​(xk​−γk​∇F(xk​)),(2)
where proxγR(⋅)\mathrm{prox}_{\gamma R} (\cdot)proxγR​(⋅) denotes the proximity operator for function R(x)R(\mathbf{x})R(x) with parameter γ\gammaγ:
proxγR(v)=argminx{R(x)+12γ∥x−v∥22}.(3)\mathrm{prox}_{\gamma R}(\mathbf{v}) = \mathop{\mathrm{argmin}}\limits_{\mathbf{x}} \left\{ R(\mathbf{x}) + \frac{1}{2\gamma} \|\mathbf{x - v}\|_2^2 \right\}. \tag{3}proxγR​(v)=xargmin​{R(x)+2γ1​∥x−v∥22​}.(3)

For linear inverse problems, we limit our scope to the following type of problems:
minx{J(x)≡12∥Ax−y∥22+λr(x)},(4)\mathop{\mathrm{min}}\limits_{\mathbf{x}} \left\{ J(\mathbf{x}) \equiv \frac{1}{2}\|\mathbf{Ax - y}\|_2^2 + \lambda r(\mathbf{x}) \right\}, \tag{4}xmin​{J(x)≡21​∥Ax−y∥22​+λr(x)},(4)
where F(x)=1/2∥Ax−y∥22F(\mathbf{x}) = 1/2\|\mathbf{Ax - y}\|_2^2F(x)=1/2∥Ax−y∥22​ is the data-fidelity term, and R(x)=λr(x)R(\mathbf{x}) = \lambda r(\mathbf{x})R(x)=λr(x) is the regularization term with parameter λ\lambdaλ.

Strictly speaking, the iterative shrinkage / thresholding algorithm (ISTA) refers to the proximal gradient algorithm for solving problem (4)(4)(4) with r(x)=∥x∥1r(\mathbf{x}) = \|\mathbf{x}\|_1r(x)=∥x∥1​. While we consider regularization terms other than the l1l_1l1​ norm, we still refer to the more generalized algorithm as ISTA.

Interpretation

Here we present an interpretation for the proximal gradient algorithm, which is helpful for further discussions. We consider the following quadratic approximation with parameter LLL:
QL(x,y):=F(y)+⟨x−y,∇F(y)⟩+L2∥x−y∥22+R(x).(5)Q_L(\mathbf{x},\mathbf{y}) := F(\mathbf{y}) + \langle \mathbf{x-y}, \nabla F(\mathbf{y}) \rangle + \frac{L}{2} \left\| \mathbf{x - y} \right\|_2^2 + R(\mathbf{x}). \tag{5}QL​(x,y):=F(y)+⟨x−y,∇F(y)⟩+2L​∥x−y∥22​+R(x).(5)
If LLL is no less than the Lipschitz constant of FFF, that is if we have L≥L(F)L \geq L(F)L≥L(F), then QL(x,y)Q_L(\mathbf{x},\mathbf{y})QL​(x,y) is a upper bound to J(x)J(\mathbf{x})J(x). In fact, we have
QL(x,y)≥J(x),∀x,andQL(y,y)=J(y).(6)Q_L(\mathbf{x},\mathbf{y}) \geq J(\mathbf{x}), \forall \mathbf{x}, \qquad \textrm{and} \qquad Q_L(\mathbf{y},\mathbf{y}) = J(\mathbf{y}). \tag{6}QL​(x,y)≥J(x),∀x,andQL​(y,y)=J(y).(6)
Therefore, QL(x,y)Q_L(\mathbf{x},\mathbf{y})QL​(x,y) is the majorizer of J(x)J(\mathbf{x})J(x), and is tight at x=y\mathbf{x = y}x=y. The minimizer of QL(x,y)Q_L(\mathbf{x},\mathbf{y})QL​(x,y) is defined as
pL(y):=argminxQL(x,y).(7)p_L(\mathbf{y}) := \mathop{\mathrm{argmin}}\limits_{\mathbf{x}} Q_L(\mathbf{x},\mathbf{y}). \tag{7}pL​(y):=xargmin​QL​(x,y).(7)
Interestingly, with some algebra, one can verify that the minimizer pL(y)p_L(\mathbf{y})pL​(y) defined above is exactly the proximal gradient update given by Eq. (2)(2)(2) with γ=1/L\gamma = 1/Lγ=1/L. That is, we have

pL(y)=argminx{R(x)+L2∥y−1L∇F(y)∥22}=proxL−1R(y−1L∇F(y)).\begin{align} p_L(\mathbf{y}) &= \mathop{\mathrm{argmin}}\limits_{\mathbf{x}} \left\{ R(\mathbf{x}) + \frac{L}{2} \left\| \mathbf{y} - \frac{1}{L} \nabla F(\mathbf{y}) \right\|_2^2 \right\} \\ &= \mathrm{prox}_{L^{-1} R} \left( \mathbf{y} - \frac{1}{L} \nabla F(\mathbf{y}) \right). \tag{8} \end{align}pL​(y)​=xargmin​{R(x)+2L​​y−L1​∇F(y)​22​}=proxL−1R​(y−L1​∇F(y)).​(8)​

Therefore, the proximal gradient algorithm can be viewed as a majorization-minimization algorithm.

Proximity operators

The proximity operator of R(x)R(\mathbf{x})R(x) is related to that of r(x)r(\mathbf{x})r(x) by
proxγR(v)=prox(γλ)r(v).(9)\mathrm{prox}_{\gamma R}(\mathbf{v}) = \mathrm{prox}_{(\gamma\lambda) r}(\mathbf{v}). \tag{9}proxγR​(v)=prox(γλ)r​(v).(9)
We consider two common choices for the penalty function r(x)r(\mathbf{x})r(x), namely the l1l_1l1​ norm and the TV norm. Both functions are not non-differentiable, and are therefore accessed through their proximity operators. The calculation for both l1l_1l1​ norm and TV norm and their corresponding proximity operators are provided by our code. It should be noted that, other choices for the regularizer are also possible.

Proximity operator for the l1l_1l1​ regularizer

If the penalty function is the l1l_1l1​ norm, i.e., r(x)=∥x∥1r(\mathbf{x}) = \|\mathbf{x}\|_1r(x)=∥x∥1​, the proximity operator for r(x)r(\mathbf{x})r(x) has a closed-form expression, which is known as the soft-thresholding operator:
(proxγr(v))i=max⁡{vi−γ,0}+min⁡{vi+γ,0},(10)\left( \mathrm{prox}_{\gamma r}(\mathbf{v}) \right)_i = \max \left\{ v_i-\gamma, 0 \right\} + \min \left\{ v_i+\gamma, 0 \right\}, \tag{10}(proxγr​(v))i​=max{vi​−γ,0}+min{vi​+γ,0},(10)
where (⋅)i(\cdot)_i(⋅)i​ denotes the iii th entry of the argument. Therefore, ISTA can be efficiently implemented for solving the problem.

Proximity operator for the TV regularizer

We consider both the isotropic and anisotropic TV norms. Unlike the l1l_1l1​ norm, for the TV regularizer r(x)=∥x∥TVr(\mathbf{x}) = \|\mathbf{x}\|_\mathrm{TV}r(x)=∥x∥TV​ in problem (3)(3)(3), there is no closed-form expression for the proximity operator. To calculate the proximity operator for the TV regularizer, one needs to solve the following denoising problem:
proxγr(v)=argminx{∥x−v∥22+2γ∥x∥TV}.(11)\mathrm{prox}_{\gamma r}(\mathbf{v}) = \mathop{\mathrm{argmin}}\limits_{\mathbf{x}} \left\{ \|\mathbf{x - v}\|_2^2 + 2\gamma\|\mathbf{x}\|_\mathrm{TV} \right\}. \tag{11}proxγr​(v)=xargmin​{∥x−v∥22​+2γ∥x∥TV​}.(11)
Here we adopt a fast iterative algorithm proposed in ref. 4 to solve this problem. The accuracy of the solution depends heavily on the number of iterations for solving this subproblem. One need to balance between efficiency and accuracy.

Line search and monotonicity

The above pseudocode presents the most basic form of these algorithms, where we have assumed that an upper bound on the Lipschitz constant L≥L(F)L\geq L(F)L≥L(F) is known, and that the calculation of pL(⋅)p_L(\cdot)pL​(⋅) is exact. Under such settings, we always have J(pL(xk))≤J(xk)J(p_L(\mathbf{x}_{k})) \leq J(\mathbf{x}_{k})J(pL​(xk​))≤J(xk​) for all kkks. Therefore, following the ISTA update with a fixed step size in Algorithm 1, the sequence {J(xk)}k\left\{ J(\mathbf{x}_k)\right\}_k{J(xk​)}k​ is non-increasing.

In practice, however, both assumptions can be violated. First, if LLL is unknown, we typically start with a small LLL and gradually increase it (that is, we start with a large step size and gradually decrease it) whenever we find our estimate is inaccurate. Second, even if we have L≥L(F)L\geq L(F)L≥L(F) already, sometimes we cannot calculate the proximity operator pL(⋅)p_L(\cdot)pL​(⋅) exactly (as is the case of TV norm), and the algorithm may converge very slowly. In this case, we would also prefer a larger LLL (i.e., a smaller step size), because we want to obtain a more accurate estimate of the proximity operator. Therefore, both problems can be addressed by line search methods. See our code for details.

Apart from line search, there is also the issue of monotonicity. Both TwIST and FISTA are not monotone algorithms, and they may sometimes diverge in numerical studies. The monotone version of both TwIST and FISTA has been presented in the original paper, and we have also implemented them in our code.

Core functions

Three core functions, namely ISTA.m, FISTA.m, and TwIST.m, implement the three algorithms, respectively.

ISTA.m

Link: ISTA.m

function [x,n_iters,J_vals,runtimes] = ISTA(y,A,lambda,...)

Required inputs:

  • y 1D / 2D / 3D array
    The observation y\mathbf{y}y.
  • A 2D array or function handle
    The forward linear operator of the system. A can be a 2D array when both x and y are 1D arrays, otherwise A should be a function handle.
  • lambda float
    Regularization parameter λ\lambdaλ. See Eq. (2).

Optional inputs:

  • 'penalty' function handle, default = @norml1
    The penalty function r(x)r(\mathbf{x})r(x), should accept one input xxx. The default choice is the l1l_1l1​ norm r(x)=∥x∥1r(\mathbf{x}) = \|\mathbf{x}\|_1r(x)=∥x∥1​.
  • 'prox_op' function handle, default = @proxl1
    Proximity operator for r(x)r(\mathbf{x})r(x), i.e., proxγr(x)\mathrm{prox}_{\gamma r}(\mathbf{x})proxγr​(x), should accept two inputs x\mathbf{x}x and γ\gammaγ. The default choice is the soft-thresholding function.
  • 'AT' function handle, default = A'
    The transpose of A\mathbf{A}A. Required when A is a function handle. When A is a 2D array, AT is automatically set to A'.
  • 'max_iter' int, default = 100
    Maximum number of iterations allowed.
  • 'min_iter' int, default = 0
    Minimum number of iterations allowed.
  • 'initializer' { 0, 1, 2, array }, default = 0
    Initializer for x\mathbf{x}x:
    • 0 →\rightarrow→ initialization at zero,
    • 1 →\rightarrow→ random initialization,
    • 2 →\rightarrow→ initialization with AT(y) or AT*y,
    • array →\rightarrow→ initialization provided by the user (same size as xxx).
  • 'stop_criterion' { 0, 1, 2, 3 }, default = 0
    Type of stopping criterion to use:
    • 0 →\rightarrow→ stop when the relative change in the objective function falls below tol,
    • 1 →\rightarrow→ stop when the relative norm of the difference between two consecutive estimates falls below tol,
    • 2 →\rightarrow→ stop when the objective function becomes equal or less than tol,
    • 3 →\rightarrow→ stop only when the number of iteration reaches max_iter.
  • 'tol' float, default = 0.01
    Tolerance, used for stopping criterion.
  • 'eta' float, default = 2
    A positive (>1) scalar used for line search.
  • 'lip' float, default = 1
    Estimate of the Lipschitz constant, which helps determine the stepsize.
  • 'verbose' bool, default = false
    Visualizes the iterative process.

Outputs:

  • x 1D / 2D / 3D array
    The reconstructed x\mathbf{x}x.
  • n_iters int
    Number of iterations.
  • J_vals 1D array
    The values of J(x)J(\mathbf{x})J(x) for each iteration.
  • runtimes 1D array
    Runtime of the algorithm for each iteration.

Example 1: LASSO regression

Link: demo_lasso.m

The LASSO problem refers to a special case of problem (4)(4)(4) with an l1l_1l1​ penalty function, i.e., r(x)=∥x∥1r(\mathbf{x}) = \left\|\mathbf{x}\right\|_1r(x)=∥x∥1​. The problem is expressed as
minx12∥Ax−y∥22+λ∥x∥1.(12)\mathop{\mathrm{min}}\limits_{\mathbf{x}} \frac{1}{2}\|\mathbf{Ax - y}\|_2^2 + \lambda \left\|\mathbf{x}\right\|_1. \tag{12}xmin​21​∥Ax−y∥22​+λ∥x∥1​.(12)
In many applications, problem (12)(12)(12) can be interpreted as finding a sparse set of coefficients x\mathbf{x}x that matches our observation y\mathbf{y}y.
Note: In many applications, we would prefer the normalized objective function that is scale invariant. In our code, we use F(x)=1/(2N)∥Ax−y∥22F(\mathbf{x}) = 1/(2N) \left\| \mathbf{Ax - y} \right\|_2^2F(x)=1/(2N)∥Ax−y∥22​ instead of F(x)=1/2∥Ax−y∥22F(\mathbf{x}) = 1/2 \left\| \mathbf{Ax - y} \right\|_2^2F(x)=1/2∥Ax−y∥22​, where NNN is the number of samples (observations).

Strong regularization promotes sparse coefficients.

We consider the image deblurring problem as follows:
minx12∥Ax−y∥22+λ∥x∥TV,(13)\mathop{\mathrm{min}}\limits_{\mathbf{x}} \frac{1}{2}\|\mathbf{Ax - y}\|_2^2 + \lambda \left\|\mathbf{x}\right\|_{\mathrm{TV}}, \tag{13}xmin​21​∥Ax−y∥22​+λ∥x∥TV​,(13)
where ∥⋅∥TV\left\| \cdot \right\|_{\mathrm{TV}}∥⋅∥TV​ can be either the isotropic or anisotropic TV norm. A\mathbf{A}A represents the blurring linear operator (e.g., convolution with a Gaussian kernel), and y\mathbf{y}y denotes the observed blurry image.
